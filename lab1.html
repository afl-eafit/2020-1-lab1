Programming Lab 1: Simulating a Pushdown Automata

TA: Juan Sebastián Cárdenas-Rodríguez

Introduction

Non-deterministic finite automata and deterministic finite automata can only represent "regular languages", hence it is important to find a model that allow for a bigger representation such as the Pushdown Automaton (PDA) [Hopcroft, Motwani and Ullman 2007]. The goal of this programming lab is to simulate a PDA.

Assignment (80%)

  • Download the GitHub repository 2019-2-lab1.

  • (60%) Using the data type in the above repository define the function:

     simPDA :: PushdownAutomata state symbol ssymbol -> [symbol] -> Bool

    that takes a PDA and a list of symbols, which then returns true if the string of symbols belong to the language of the PDA.

    Suggestion: Use the Stack data type.

  • (10%)
    • Your lab will be tested using the -Wall, -Wmissing-local-signatures and -Wmissing-signatures options.
    • Annotate (in English) your source code using Haddock. Documentation should explain your solution for the assignment.
    • Improve your implementation with HLint.

  • (10%) Solve and deliver on time the proposed warm-up.

  • The programming lab may either be solved on your own, or jointly with one other student taking the course.

  • You are to use GHCi ≥ 8.6.5. See installing a proper Haskell distribution.

Examples

Using the PushdownAutomata.hs module in the above GitHub repository, this is what a sample input-output looks like:

      GHCi> simPFA example0 []
      False
      GHCi> simPFA example0 [0,1,1,0]
      True
      GHCi> simPFA example0 [0,1,1,1,0,0,1,1,1,0]
      True
      GHCi> simPFA example1 "aaaab"
      False
      GHCi> simPFA example1 "aabb"
      True
      GHCi> simPFA example2 "aaaaaabbbbbbbbb"
      True
